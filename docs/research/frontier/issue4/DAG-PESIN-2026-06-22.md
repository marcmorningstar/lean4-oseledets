# DAG — Wall #10: the Pesin / Mañé / Ledrappier–Young tower behind `∑λ⁺ ≤ h`

**Target leaf:** `Frontier.Issue4Pesin.sumPosExp_le_ksEntropy_of_SRB`
(`Frontier/Issue4Pesin/ManeLowerBound.lean`), the **reverse** Pesin inequality
`(sumPosExp : EReal) ≤ h_μ(T)`. Combined with the proved Margulis–Ruelle sharp inequality
(`Oseledets.margulisRuelle_sharp`) this gives Pesin's formula by `le_antisymm`.

**Date:** 2026-06-22. **Mode:** cartography only — no `.lean` edits, no builds, no git.
**Method:** read the saved references + prior feasibility report; grepped Mathlib
(`.lake/packages/mathlib/Mathlib`) and `Oseledets/` for every node to set `mathlibStatus` by name.

**Headline verdict:** This is a **deep, narrow** wall — the opposite shape from the main Oseledets
MET (which was wide-and-shallow on existing Mathlib). The critical path is dominated by **Layer 2
(nonuniform hyperbolicity / Pesin geometry)**, not by Layer 1 (SMB / countable-partition entropy).
Layer 2 must build, in serial order, an entire Pesin-theory sub-library that **does not exist in any
form in Mathlib**: a measurable `Eˢ⊕Eᵘ` splitting, the stable/unstable manifold theorem, the
unstable foliation and **its absolute continuity**, and leaf disintegration. Each of these is a
high-design-risk new abstraction that gates everything below it. Layer 1 (SMB + a countable-partition
KS-entropy API) is substantial but is *ordinary ergodic theory on existing Mathlib martingale +
project Birkhoff/Kingman*, and runs **in parallel** with Layer 2 — it is not on the critical path.

## 0. Two-layer overview, as actually structured in the leaf

The leaf already decomposes the wall top-down into exactly two BLOCKED leaves plus sorry-free
plumbing:

- **Capstone** `sumPosExp_le_ksEntropy_of_SRB` — **sorry-free assembly** chaining the two layers:
  `sumPosExp = ∫χ ≤ ∫h_ν ≤ h_μ(T)`. The first `=` is L0 (a.e.-constant integrand over a probability
  measure, already discharged), the middle `≤` is `integral_mono_ae` of Layer 2's pointwise bound,
  the final `≤` is Layer 1. **This node is DONE** — no new work.
- **`manePropLowerBound`** (Layer 1, Contractor Prop 7.7): `∫ h_ν dμ ≤ h_μ(T)` + integrability.
- **`localEntropy_ge_unstableJacobian`** (Layer 2, Contractor Thm 7.15 / (7.17)/(7.18)):
  `χ ≤ h_ν` a.e.
- **`rate_log_div_tendsto_zero_of_tendsto_pos`** (L1b, Prop 7.7 step 3): **already proved sorry-free**
  in the leaf — the `(1/n)·log(μ(Pₙ)/ν(Pₙ)) → 0` real-analytic core.
- **`localNuEntropy`** — currently an `opaque` real function (a black box). Making it a *real*
  definition (the limsup shrink-rate of a genuine leaf measure `ν` along a genuine dynamical
  refinement `Sₙ`) is itself a **high-design-risk** node that both layers depend on (see N-LNE).

So the two genuine sorries are `manePropLowerBound` and `localEntropy_ge_unstableJacobian`, and the
real engineering question is the dependency tree *below* each.

## 3. Critical path, depth, width

**Critical path (longest must-build-in-order chain of ABSENT nodes)** runs entirely through Layer 2,
because each Pesin geometric object is the input to the next:

```
N-L2-SPLIT → N-L2-CHARTS → N-L2-GT → N-L2-WU-MFD → N-L2-WU-FOL → N-L2-DISINT → N-SRB → N-L2-THM7.15 → N-CAP
```

**Depth = 9 absent nodes** on this serial chain (N-CAP itself is done, but the chain ends in it; the
8 absent predecessors plus N-L2-THM7.15 = 9 absent nodes that cannot be parallelised away). Every one
of the eight Layer-2 geometric nodes is **designRisk high** — this is the defining feature of the
wall: it is not LOC-bound, it is *abstraction-bound*. Each node's correct Lean statement is itself a
research question (what is the right Lean type for a measurable foliation? for its absolute
continuity? for leaf disintegration of a non-σ-finite partition?), and getting it wrong ripples
downstream.

Layer 1 forms a **second, parallel chain** of depth ~4
(`N-L1-CTBL-API → N-L1-7.6 → {N-L1-SMB, N-L1-RN} → N-L1-PROP7.7`) that is **shorter** than Layer 2 and
runs concurrently, so it does **not** extend the critical path. Hence **Layer 2 (Pesin) dominates the
critical path, decisively**; SMB is the hardest *Layer-1* node but Layer 1 as a whole is off the
critical path.

**Max parallel width** at the widest absent level ≈ **6**: once `N-L2-SPLIT` and `N-L1-CTBL-API` (the
two roots) exist, the following can be attacked simultaneously: `N-L2-CHARTS`, `N-L2-RETURN`,
`N-L1-7.5/7.6`, `N-L1-SMB`, `N-L1-RN`, `N-L3-POWER`. But this width is *deceptive*: the high-designRisk
Layer-2 nodes are mostly serial (each gates the next), so parallelism cannot compress the dominant
chain — the wall is **deep, not wide**.

## 4. Two clocks

### (a) Human / community Mathlib-PR cadence
Each Layer-2 node (measurable splitting, Pesin charts, graph transform, unstable manifold theorem,
foliation absolute continuity, leaf disintegration, SRB a.c.) is, historically and at Mathlib review
cadence, a **multi-month-to-multi-year** contribution in its own right — the unstable-manifold
theorem and the absolute continuity of the foliation are each PhD-thesis-scale formalization targets
with no existing Mathlib scaffolding. Layer 1 (countable-partition KS entropy + SMB) is a clean but
substantial **multi-month** development. Realistically this is a **multi-year** community effort
(≈ 3–6 years to a full sorry-free close, dominated by the Pesin sub-library), comparable to standing
up an entire `Mathlib/Dynamics/Pesin` area from zero.

### (b) Warm-worktree parallel-Claude regime
Wall-clock ≈ critical-path-depth × per-node (design + prove + integrate), since breadth is absorbed
by running many warm agents at once. Calibration anchor from this project: a bounded, well-specified
lemma against existing Mathlib closes in ~one warm session; a **high-design-risk new-abstraction
node** costs several iterations and gates everything downstream.

Here the critical path is **9 nodes, eight of them high-design-risk *new abstractions with no Mathlib
prior*** — the worst case for the compression trick, because the parallel-agent advantage (width)
does nothing for a serial chain of design-novel nodes, and each node's *design* (not just its proof)
must be discovered. Budgeting roughly **1–2 weeks of warm staged campaigning per high-design-risk
Layer-2 node** (design the type, prototype, prove, integrate, audit) — these are genuinely
research-level abstractions, not Mathlib-lookups — the serial chain alone is **~3–5 months of
wall-clock**, even with unlimited parallel width, because the depth cannot be parallelised and the
per-node cost is design-dominated. Layer 1 (off critical path) finishes inside that window.

Contrast with the main Oseledets MET, which compressed "many months" → ~3 days precisely because it
was **breadth on existing Mathlib** (wide, shallow, low-design-risk). Wall #10 is the inverse: depth
on *absent* foundations, high design risk per node. The compression factor here is therefore **small
on the critical path** (~weeks → still weeks per node; the gain is only that the wide Layer-1 work and
the within-Layer-2 side-nodes parallelise away).

**Reachable-now subset (does NOT touch the critical path), worth landing independently:**
`N-L1-RN` (martingale identification; engine exists), `N-L2-RETURN` (Birkhoff return-time; engine
exists in project) — each ~1 warm session given an interface; and the `N-L1-CTBL-API` +
`N-L1-SMB` chain as the "hard but not research-wall" multi-week Layer-1 keystone. These convert real
content but leave the Layer-2 critical path intact.

## 5. Biggest risks (design-risk, in critical-path order)

1. **N-L2-WU-FOL — absolute continuity of the unstable foliation.** The single deepest node;
   historically the hardest part of Ledrappier–Young. No Mathlib prior for foliations *or* holonomy
   absolute continuity. The right Lean type for "measurable lamination whose holonomy preserves null
   sets" is itself unresolved.
2. **N-L2-WU-MFD — Pesin (nonuniform) stable/unstable manifold theorem.** Hadamard–Perron in the
   tempered setting; a major sub-library; gates the foliation and the leaf measure.
3. **N-L2-DISINT — disintegration of `μ` along `Wᵘ`.** Mathlib's `condKernel` disintegrates along a
   standard-Borel measurable map, **not** a foliation; bridging that gap (non-σ-finite leaf partition)
   is an open design problem and gates `N-SRB`.
4. **N-L2-SPLIT / N-L2-CHARTS — measurable Pesin block + charts.** The root abstractions of the whole
   Layer-2 chain; every downstream node inherits their type choices.
5. **N-LNE — making `localNuEntropy` a real definition.** Currently `opaque`; both layers reference
   it, so its type ripples into N-L1-PROP7.7 *and* N-L2-THM7.15. Cannot be finalised until the leaf
   measure `ν` (N-L2-WU-LEAF-MEAS) exists — a hidden cross-layer coupling.
6. **N-L1-SMB — Shannon–McMillan–Breiman.** The largest *non-Pesin* gap; off the critical path but a
   multi-week keystone unblocking all of Layer 1; high design risk in the countable-partition entropy
   formulation it rests on (N-L1-CTBL-API).

## 6. Bottom line

- **Which layer dominates the critical path:** **Layer 2 (Pesin geometry)**, unambiguously. Layer 1
  (SMB + countable-partition entropy) is hard but *parallel and shorter*; SMB is the hardest Layer-1
  node yet still off the critical path.
- **Shape:** deep (depth 9), narrow (the dominant chain is serial), high design-risk per node — the
  inverse of the wide-shallow MET that compressed to days.
- **Compression:** the parallel-Claude regime helps mainly by absorbing Layer 1 and the Layer-2
  side-nodes; the serial, design-novel Pesin chain is **~3–5 months wall-clock** even with unlimited
  agents, vs. a **multi-year** community effort. A full sorry-free close requires standing up a
  `Pesin`/foliation/leaf-disintegration sub-library that today has **zero** Mathlib precursors.
