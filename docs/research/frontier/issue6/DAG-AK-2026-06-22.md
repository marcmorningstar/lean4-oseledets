## 4. Critical path & width

### Residual (what this project must actually do)

Critical path (all ABSENT, strictly serial — each is the *only* dependency of the next):

```
R-coanalytic-class  →  R-reduction-465  →  R-structure-471  →  R-openSection-472
```

- **criticalPathDepth = 4** absent nodes.
- **maxParallelWidth = 1.** The residual chain is a pure line — there is *no* breadth to absorb with
  parallel agents. This is the structural opposite of the main Oseledets MET (wide & shallow): here
  it is **narrow & deep**, and the two upstream nodes (`R-coanalytic-class`, `R-reduction-465`) are
  high-design-risk *new abstractions* that gate everything.
- Mathlib leaves the chain rests on: `AnalyticSet`, `AnalyticSet.measurablySeparable`,
  `AnalyticSet.measurableSet_of_compl` (Suslin), `MeasurableSet.analyticSet_image`,

  is genuinely already in Mathlib, which is what makes 4 (not 10) the depth.

`{H-compact-borel, D-derivative, WO-codings, P-reflection, R-reduction-465}` — width ≈ 5 — but the

## 5. Effort estimate (two clocks)

### Residual chain (the honest target)

**Critical-path depth 4, width 1, two high-design-risk nodes.**

- `R-coanalytic-class` (high design risk): the *right* Lean encoding of the coanalytic pointclass is
  itself the question — as a predicate `IsCoanalytic := AnalyticSet ∘ compl`, vs. a Σ/Π hierarchy
  framework. The cheap encoding (`compl` of analytic) is enough for 4.6.5 and is the pragmatic bet;
  designing it so 4.6.5/4.7.1 go through is the risk. ≈ 2–4 warm-iteration sessions.
- `R-reduction-465` (high design risk): the generalized reduction theorem for a *sequence* of
  coanalytic sets with Borel union, built by transfinite/inductive use of
  `AnalyticSet.measurablySeparable`. This is the heart; the inductive statement must be stated so the
  union is preserved. ≈ 3–5 sessions, and it gates the two below.
- `R-structure-471` (med): 4.7.1 from 4.6.5 — disjoint-analytic-with-closed-section ⟹ Borel
  separator, base-element bookkeeping. ≈ 1–2 sessions.
- `R-openSection-472` (med, = the leaf): assemble `C = ⋃ₙ (A n ×ˢ V n)` with `A n = {x | Vₙ ⊆ Cₓ}`
  replaced by a Borel `A n` from 4.7.1; the set-theoretic identity is elementary. ≈ 1 session given
  the above.

**parallelWorkflowWallClock:** ≈ depth 4 × (1–4 sessions) with the two upstream design-risk nodes
dominating ⇒ **~2–4 weeks of staged warm-worktree campaigns**, essentially serial because width = 1.
Parallelism buys little here; the leverage is instead in *de-risking the upstream definition early*
(spike `R-coanalytic-class` + `R-reduction-465` before committing to 4.7.1/4.7.2).

**humanCommunityCalendar:** at Mathlib-PR cadence, a coanalytic-pointclass + generalized-reduction
contribution is a **multi-month (~3–6 month), 2–4 PR** effort (new pointclass file, reduction
theorem, structure theorem, the projection corollary), gated on reviewer consensus about the *right*
coanalytic API — the same design debate that makes the nodes high-risk.

### Full general-Y tower (NOT required)

Depth ≈ 7 with two more high-risk new-abstraction nodes (`H-effros-space`, `D-rank/derivative`,
`P-boundedness`, `WO-codings`). Parallel regime ~**2–4 months** of staged campaigns; community cadence
**1–2+ years** across ~6–8 PRs. No proof assistant has formalized Effros-standard-Borel-ness,
Π¹₁-boundedness, or Arsenin–Kunugui (confirmed: arXiv 2405.16603 lists `F(P)` Effros as a known
*unformalized* object). This is the wall the feasibility report named — correctly — but the repo
routed around it.

## 6. Biggest risks

1. **Design risk in `R-coanalytic-class`/`R-reduction-465` (width-1 chain).** With no parallel
   breadth, a wrong coanalytic encoding ripples through 4.7.1/4.7.2 and forces a restate-and-redo of
   the whole residual chain. The single highest-leverage action is an early throwaway spike fixing
   the coanalytic API before building on it.
2. **Mathlib has the *two-set* first separation but not the *sequence* reduction.** It is tempting to
   assume 4.6.5 is "just iterating `measurablySeparable`"; it is not — preserving the Borel union
   over a countable family with the reduction property is the real content and the main proving cost.
3. **Over-scoping to the §5.12 tower.** The feasibility report's "research wall" verdict is true for
   *general-Y* AK but **stale for the residual**: re-deriving Effros/Π¹₁-boundedness would be months
   of work the consumer does not need. Keep the ball-slab elimination; do **not** rebuild §5.12.
4. **`Closeds α` looks like Effros but is not** (Hausdorff/Vietoris topology; correct Borel structure
   only when `Y` compact, and no `MeasurableSpace` instance). Any agent reaching for it for a general
   Effros space will get a subtly wrong object.
5. **The FALSE-shortcut counterexample is load-bearing** (`not_isClosed_image_fst_…`): it proves the
   compact-section hypothesis cannot be dropped and the projection is only Borel (`F_σ`), not closed —
   so no topological one-liner can replace 4.7.2.
